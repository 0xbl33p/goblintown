User dashboard

This is the future Goblintown account surface for userland DB state. It should be signed-in, opt-in, and separate from the ChatGPT app widget.

Account

Sign in, cloud mode, friend code, country metadata, and account status.

Warrens

Connected Warrens, local/cloud sync status, recent rites, and Hoard summaries.

Settings

Provider preferences, sentiment source keys, add-ons, mail, and reset controls.

Artifacts

Typed Artifact history, imports, embeddings/vectorization state, and lineage links.

Boundary

  • ChatGPT-hosted rites do not require or spend local provider keys.
  • Userland DB storage should require sign-in and explicit user consent.
  • Local-only secrets stay local unless the user chooses cloud mode.
